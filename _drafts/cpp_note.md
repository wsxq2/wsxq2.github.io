
## 概述

C++编程语言是 C 语言是 Plus 版本，之所以不是 C+ 而是 C++，我猜想是因为相比于 C 语言添加的特性不足以使用一个加号来表达，所以选择使用两个加号。在学 C++ 前，我以为 C++ 会过于复杂，难以入门，而事实上，只要对 C 语言熟悉到一定程度，学习 C++ 并不困难。C++ 的许多特性是非常自然的，容易接受的，但由于它是一个古老的语言，所以为了兼容性做了许多让步，保留了许多看起来不够简洁的特性。但兼容性具有不可忽视的重要性，任意语言刚出来时都是最简洁的时候，随着它的演进，总会变得更加复杂，如果不考虑兼容性，那就是可怕的灾难。不说别的，Python2 到 Pyhton3 的演进就是一个典型的例子，由于二者不兼容，产生许多恶心的问题。

C++ 是很有意思的语言，我觉得每个程序员都得学习一下此语言。我记得曾有人说过，编程语言是无穷无尽的，是否学习一门编程语言取决于它是否有非常不一样的地方，当然，热度也是不可忽略的因素。对于笔者而言，主要学习了 C、Java、Python，最后才学习得 C++。学习这几门语言的原因大致如下：C 语言不必多说，号称入门计算机必学的编程语言；而 Java 引入了虚拟机，且是纯粹的面向对象的语言，和 C 有极大的区别，所以也学习了；Python 是一门解释型语言，其简练而友好的语法，以及大量的库引入的强大的功能具有极大的吸引力，从各种意义上来说，都和 C 语言具有显著的差异，因此也学习了。那么 C++ 呢？C++ 一方面是 C 的升级版，学了 C 不学 C++ 总有点没学全的感觉；另一方面，C++ 也有许多强大的特性，包括面对对象、泛式编程、重载运算符等，极具吸引力。

有人说，语言只是工具，背后的思想才是最重要的，比如面向过程的编程思想、面向对象的编程思想、异步思想、轮询、事件驱动、状态机等。我觉得这是非常有道理的，让我印象深刻的是异步思想，我在学习 JavaScript 时，其强调它使用了异步思想，即只有一个线程，但同样能处理多个任务，我当时学得似懂非懂，但后来接触单片机后，发现单片机中也有类似思想的应用，这是因为单片机通常只有一个线程，因为考虑到实时性要求，就不能写任何阻塞式代码，那么怎么解决呢？就可以使用异步思想，注入一个回调函数，调用时传入此回调函数，并且立即返回，原函数中注册此回调函数，在合适的时机调用它。（等一下，单片机中似乎不太一样，它通常使用状态机+缓存的方式解决此问题，比如要从某个驱动器读取状态，此时会有一个轮询的状态机，在合适的状态下读取数据到模块的全局变量中，然后读取数据时直接从缓存中获取。写入时也类似，调用写入函数时，只是临时写入到缓存变量，待状态机执行时才真正写入到驱动器）

本文是对 C++ 相关资源、个人使用经验的总结，不是一个入门教程。

## 个人经验

- `const JointStateMap& states` 和 `JointStateMap const & states` 在 C++ 中等价，`int const * p` 或 `const int * p`也等价，表示`*p`不可修改，但`int * const p`和它们不同，表示`p`本身不可修改。读法技巧：把 `const` 看作紧跟它右边的对象
